# 🤖 AI Database Analyst Bot

> Интеллектуальный Telegram-бот, превращающий естественный язык в аналитику баз данных. Использует LLM для маршрутизации намерений, генерации SQL и автоматического исправления запросов при ошибках.

### 💡 Суть проекта

Бот решает проблему "порога входа" в базы данных. Пользователю не нужно знать SQL, структуру таблиц или писать сложные запросы. Он просто пишет обычное предложение в Telegram, а бот сам определяет намерение, извлекает данные, анализирует их и возвращает понятный ответ, таблицу или диаграмму.

---

### 🚀 Ключевые архитектурные особенности

- 🧠 **Маршрутизация намерений (Intent Routing):** Бот не требует жестких команд (кроме `/start`). Легковесный промпт-классификатор на базе LLM анализирует текст и автоматически выбирает один из 4 сценариев работы.
- 🩹 **Self-Healing SQL (Самовосстановление):** При генерации SQL-запроса бот сначала валидирует его, а затем выполняет. Если база данных возвращает ошибку, бот анализирует текст ошибки, корректирует запрос и повторяет попытку (до 3 итераций).
- 🗣️ **Контекстная память (Sliding Window):** Реализована через хранение последних 3 пар сообщений (вопрос-ответ) с инъекцией префикса "История диалога". Это дает LLM понимание контекста без перегрузки токенами.
- 🌐 **Мульти-провайдерность:** Архитектура позволяет гибко переключаться между облачными API (OpenAI, OpenRouter) и локальными моделями (LM Studio) для баланса между скоростью, стоимостью и приватностью.

---

### 🎯 Режимы работы (Сценарии)

1. **💬 Обычный диалог:** Ответы на общие вопросы и уточнения, не требующие обращения к БД.
2. **📊 Генерация ER-диаграмм:** Построение визуальных связей между таблицами (как полной схемы, так и по запросу конкретных сущностей).
3. **SQL-запросы (Data Retrieval):** Точное извлечение данных из БД по текстовому вопросу.
4. **Аналитика:** Выполнение SQL-запроса, получение сырых данных и их последующий семантический анализ LLM с выдачей структурированного текстового ответа.

### 🛠️ Технологический стек и Архитектура

- **Язык:** Python
- **Telegram API:** `aiogram` (асинхронный клиент)
- **База данных:** `SQLite`
- **ИИ и маршрутизация:** `OpenAI API`, `OpenRouter`, `LM Studio` (локальные модели)
- **Архитектурные нюансы:** Бот построен на асинхронном ядре `aiogram`. На текущем этапе вызовы к внешним LLM API и БД реализованы синхронно внутри асинхронных хендлеров (требует дальнейшей оптимизации через `asyncio.to_thread` или асинхронные клиенты для LLM).
